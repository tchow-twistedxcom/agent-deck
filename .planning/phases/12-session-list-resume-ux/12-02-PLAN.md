CRITICAL anti-patterns to avoid (from research):
- Do NOT call `UpdateClaudeSessionsWithDedup` inside `Restart()` itself. `Restart()` only has access to a single instance, not the full `[]*Instance` slice. The dedup function needs the full slice to detect cross-session conflicts. The call belongs in the UI layer.
- Do NOT pass a single-element slice like `[]*Instance{current}` to the dedup function. This defeats the purpose since there's only one element to compare. Always pass `h.instances` (the full slice under `h.instancesMu` lock).

DEDUP-01 verification: `Restart()` already mutates the existing `*Instance` in place (never creates a new row). package session

```go
import (
    "path/filepath"
    "sync"
    "testing"
    "time"

    "github.com/asheshgoplani/agent-deck/internal/statedb"
    "github.com/stretchr/testify/assert"
    "github.com/stretchr/testify/require"
)

func TestConcurrentStorageWrites(t *testing.T) {
    // 1. Create a temp dir with a single state.db
    tmpDir := t.TempDir()
    dbPath := filepath.Join(tmpDir, "state.db")

    // Helper: open a Storage against the shared path
    openStorage := func() *Storage {
        db, err := statedb.Open(dbPath)
        require.NoError(t, err)
        require.NoError(t, db.Migrate())
        t.Cleanup(func() { db.Close() })
        return &Storage{db: db, dbPath: dbPath, profile: "_test"}
    }

    s1 := openStorage()
    s2 := openStorage()

    sharedClaudeID := "claude-session-shared-123"

    // 2. Both create sessions with the same ClaudeSessionID
    instances1 := []*Instance{{
        ID:               "sess-from-s1",
        Title:            "S1 Session",
        ProjectPath:      "/tmp/s1",
        GroupPath:        "test",
        Command:          "claude",
        Tool:             "claude",
        Status:           StatusRunning,
        ClaudeSessionID:  sharedClaudeID,
        CreatedAt:        time.Now().Add(-1 * time.Minute),  // older
    }}

    instances2 := []*Instance{{
        ID:               "sess-from-s2",
        Title:            "S2 Session",
        ProjectPath:      "/tmp/s2",
        GroupPath:        "test",
        Command:          "claude",
        Tool:             "claude",
        Status:           StatusRunning,
        ClaudeSessionID:  sharedClaudeID,
        CreatedAt:        time.Now(),  // newer
    }}

    // 3. Write concurrently
    var wg sync.WaitGroup
    var err1, err2 error

    wg.Add(2)
    go func() {
        defer wg.Done()
        gt1 := NewGroupTree()
        gt1.AddSession(instances1[0])
        err1 = s1.SaveWithGroups(instances1, gt1)
    }()
    go func() {
        defer wg.Done()
        gt2 := NewGroupTree()
        gt2.AddSession(instances2[0])
        err2 = s2.SaveWithGroups(instances2, gt2)
    }()
    wg.Wait()

    // Both writes should succeed (SQLite WAL mode allows concurrent access)
    require.NoError(t, err1, "s1 SaveWithGroups should succeed")
    require.NoError(t, err2, "s2 SaveWithGroups should succeed")

    // 4. Load from a third storage and verify dedup
    s3 := openStorage()
    loaded, _, err := s3.Load()
    require.NoError(t, err)

    // Count how many sessions retained the shared ClaudeSessionID
    holdersCount := 0
    for _, inst := range loaded {
        if inst.ClaudeSessionID == sharedClaudeID {
            holdersCount++
        }
    }
    // After dedup in SaveWithGroups, at most one session should hold the shared ID
    assert.LessOrEqual(t, holdersCount, 1,
        "At most one session should retain the shared ClaudeSessionID after dedup")
}
```

Key patterns:
- Use `statedb.Open(dbPath)` with the SAME `dbPath` for all three storage instances (tests WAL concurrent access)
- Use `t.Cleanup(func() { db.Close() })` for each db (not `defer`, which doesn't survive subtests)
- Use `sync.WaitGroup` for concurrent writes
- Use `NewGroupTree()` and `AddSession()` for creating valid group trees
- The `SaveWithGroups` function already calls `UpdateClaudeSessionsWithDedup` internally, so each individual save deduplicates its own slice. The key assertion is that loading all sessions from DB after both saves yields at most one holder of the shared ID.
- Ensure `time.Now().Add(-1 * time.Minute)` for older session to establish deterministic dedup winner.

Do NOT use `newTestStorage(t)` directly because it creates separate temp dirs. The test needs a shared `dbPath`.
