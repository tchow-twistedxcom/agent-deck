<objective>
Fix non-interactive auto-start on WSL/Linux by adding pane-ready detection before SendKeysAndEnter and replacing shell uuidgen with Go-side UUID generation.

Purpose: Resolves PLAT-01 (#311). Two root causes are addressed: (1) Mode A timing race where send-keys fires before the pane shell is ready on slow WSL2 cold-starts, and (2) Mode E where uuidgen may not be installed on minimal Linux/WSL, causing empty session IDs and triggering the problematic double bash -c fish-compat wrap.

Output: New `internal/tmux/pane_ready.go` + tests, modified `internal/tmux/tmux.go` Start(), modified `internal/session/instance.go` (generateUUID + 3 call sites), updated tests.
</objective>

    ```go
    func generateUUID() string {
        b := make([]byte, 16)
        if _, err := rand.Read(b); err != nil {
            return fmt.Sprintf("00000000-0000-0000-%016x", time.Now().UnixNano())
        }
        b[6] = (b[6] & 0x0f) | 0x40 // version 4
        b[8] = (b[8] & 0x3f) | 0x80 // variant bits
        return fmt.Sprintf("%08x-%04x-%04x-%04x-%012x",
            b[0:4], b[4:6], b[6:8], b[8:10], b[10:16])
    }
    ```
