<objective>
Build the watcher engine that orchestrates the full event pipeline: adapter goroutines produce Events, a single-writer goroutine serializes DB writes via a buffered channel, dedup is handled by INSERT OR IGNORE, the router determines event routing, and health trackers are updated after each write. Engine Stop() cancels all adapter contexts cleanly with zero goroutine leaks.

Purpose: This is the runtime core that ties together all Plan 01 components (adapter interface, router, health tracker) into a working event-processing pipeline.
Output: engine.go with full lifecycle, mock adapter for testing, comprehensive test suite including goleak verification.

6. Implement `(e *Engine) Start() error`:
   - For each adapter entry:
     a. Call adapter.Setup(e.ctx, entry.config). If error, log and skip this adapter.
     b. Create derived context: `adapterCtx, adapterCancel := context.WithCancel(e.ctx)`. Store adapterCancel in entry.
     c. e.wg.Add(1), launch goroutine: `go e.runAdapter(adapterCtx, &entry)`.
   - e.wg.Add(1), launch single-writer goroutine: `go e.writerLoop()`.
   - If cfg.HealthCheckInterval > 0: e.wg.Add(1), launch health check goroutine: `go e.healthLoop()`.
   - Return nil.

7. Implement `(e *Engine) runAdapter(ctx context.Context, entry *adapterEntry)`:
   - defer e.wg.Done()
   - Call entry.adapter.Listen(ctx, e.eventCh). If error and ctx not cancelled, log the error and call entry.tracker.RecordError().

8. Implement `(e *Engine) writerLoop()` per D-13:
   - defer e.wg.Done()
   - for loop with select:
     - case event from e.eventCh:
       a. Find the adapter entry by event.Source (match on config.Type; if multiple adapters of same type, use first match or iterate all -- for Phase 13, source matching is sufficient since watcher names are unique).
       b. Route: call e.cfg.Router.Match(event.Sender). Get RouteResult (may be nil).
       c. Determine routedTo: if result != nil, routedTo = result.Conductor, else routedTo = "" (per D-08).
       d. Call e.cfg.DB.SaveWatcherEvent(entry.watcherID, event.DedupKey(), event.Sender, event.Subject, routedTo, "", e.cfg.MaxEventsPerWatcher) per D-10.
       e. If inserted (bool true): call entry.tracker.RecordEvent(). Non-blocking send event to e.routedEventCh. Per D-14.
       f. If error from SaveWatcherEvent: call entry.tracker.RecordError(). Log error.
     - case <-e.ctx.Done(): return

   NOTE: The writerLoop needs to know which adapter produced each event. Add a `watcherID` field to Event struct (internal, not exported via JSON -- use `json:"-"` tag). Each runAdapter goroutine sets event.watcherID before sending to eventCh. Actually, to avoid modifying the Event struct (which is a public contract), create an internal `eventEnvelope` struct in engine.go with `event Event` and `watcherID string`. The eventCh becomes `chan eventEnvelope` internally. runAdapter wraps each event in an envelope before sending.

   REVISED approach: Use `eventEnvelope` struct:
   ```go
   type eventEnvelope struct {
       event     Event
       watcherID string
       tracker   *HealthTracker
   }
   ```
   - eventCh becomes `chan eventEnvelope` (capacity 64)
   - runAdapter wraps: `e.eventCh <- eventEnvelope{event: evt, watcherID: entry.watcherID, tracker: entry.tracker}`
   - writerLoop reads envelopes, uses envelope.watcherID and envelope.tracker directly

9. Implement `(e *Engine) healthLoop()`:
   - defer e.wg.Done()
   - ticker := time.NewTicker(e.cfg.HealthCheckInterval)
   - defer ticker.Stop()
   - for loop with select:
     - case <-ticker.C: for each adapter entry, call entry.adapter.HealthCheck(). If error, entry.tracker.SetAdapterHealth(false) + entry.tracker.RecordError(). Else entry.tracker.SetAdapterHealth(true). Then state := entry.tracker.Check(). Non-blocking send state to e.healthCh.
     - case <-e.ctx.Done(): return

10. Implement `(e *Engine) Stop()` per D-18:
    - Call e.cancel() (cancels root context, propagates to all derived adapter contexts).
    - For each adapter, call adapter.Teardown() (best-effort, log errors).
    - e.wg.Wait() (waits for all goroutines to exit).

11. Implement `(e *Engine) EventCh() <-chan Event`:
    - Return e.routedEventCh (read-only channel per D-20).

12. Implement `(e *Engine) HealthCh() <-chan HealthState`:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| adapter -> engine eventCh | Adapters push events to engine's internal channel; malformed events could cause panics |
| engine -> statedb | Engine calls SaveWatcherEvent; SQL injection mitigated by parameterized queries in statedb |
| engine -> routedEventCh | Routed events exposed to TUI; contains sender/subject metadata |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-06 | D (Denial of Service) | eventCh buffer | mitigate | Buffered channel capacity 64; runAdapter uses non-blocking send to prevent adapter goroutine hang if channel full; dropped events logged |
| T-13-07 | T (Tampering) | Event.DedupKey | accept | DedupKey is sha256 of event fields; collision is astronomically unlikely; acceptable for dedup purposes |
| T-13-08 | E (Elevation of Privilege) | Engine.Stop | mitigate | Stop() calls cancel() first (stops all work), then Teardown (cleanup); wg.Wait ensures no orphan goroutines; goleak test enforces in CI |
| T-13-09 | R (Repudiation) | Event processing | mitigate | All events persisted to watcher_events table with timestamps before routing; provides audit trail |
| T-13-10 | D (Denial of Service) | writerLoop single goroutine | accept | Single-writer is intentional bottleneck to prevent SQLite contention; channel buffer absorbs bursts; at 64 events/sec throughput this is sufficient for the expected event rate (few per minute) |
</threat_model>
