   - `mu sync.RWMutex`

   - `eventTimestamps []time.Time` (sliding window for rolling rate, keep only last hour)

   - Lock mu, increment consecutiveErrors.

   - Lock mu, set adapterHealthy.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem -> Router | clients.json loaded from disk; could be malformed or tampered |
| adapter -> Event | Adapters produce Event structs; engine trusts normalized fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | T (Tampering) | router.go LoadClientsJSON | mitigate | Validate JSON schema on load; reject entries with empty conductor field; log warning for unexpected keys |
| T-13-02 | I (Info Disclosure) | Event.Body | accept | Body contains message content by design; access controlled at filesystem level (meta.json perms 0644) |
| T-13-03 | D (Denial of Service) | HealthTracker.eventTimestamps | mitigate | Lazy prune timestamps older than 1 hour on RecordEvent; prevents unbounded memory growth |
| T-13-04 | S (Spoofing) | Event.Sender | accept | Sender normalization is adapter responsibility (Phase 14+); router trusts normalized sender string |
| T-13-05 | T (Tampering) | clients.json path | mitigate | LoadFromWatcherDir uses WatcherDir() (hardcoded ~/.agent-deck/watchers); no user-controlled path segments in this phase |
</threat_model>
